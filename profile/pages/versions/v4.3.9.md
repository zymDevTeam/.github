# Lanzamiento de Versión 4.3.9

## General
* fix: Estandarización del manejo de `custom_attributes` usando `transformMapToFirestoreDoc` y encadenamiento opcional.
* Mejora en el formato y consistencia de toda la documentación de los módulos.
* chore: Eliminación del archivo `package-lock.json` del repositorio.
* feat: Se agregó `vitepress-plugin-mermaid` y sus dependencias requeridas en la configuración del proyecto.

## POQTROL
* refactor: Simplificación de la lógica de validación en `DynamicFormComponent`.
* fix: Actualización del formato de datos en la generación de reportes para mayor consistencia.
* fix: Actualización de la nomenclatura de instrumentos en las tablas de rendimiento y estadísticas para mayor claridad.
* feat: Implementación del seguimiento de auditoría en la generación de estadísticas para POQTROL.
* feat: Se agregó la funcionalidad de seguimiento de auditoría y exportación para las estadísticas de POQTROL.
* Documentación técnica para el flujo de procesamiento de estadísticas en Quattrol365:
  - Se agregó un nuevo archivo que detalla el flujo técnico y lógico para el procesamiento de datos estadísticos dentro del sistema Quattrol365. Describe los roles de los componentes PreStatsGenerator y StatsGenerator, sus responsabilidades y los cálculos estadísticos realizados, incluyendo métodos de filtrado y generación de métricas.
* feat: Documentación estadística de POQTROL para usuarios de glucómetro y cualitativo:
  - Se creó `stats_POQTROL_glucometer_user.md` detallando el procesamiento técnico para generar estadísticas individuales en eventos de glucometría, incluyendo extracción de datos del usuario, indicadores de rendimiento e integración de metadatos.
  - Se creó `stats_POQTROL_qualitative_user.md` describiendo el procesamiento técnico para generar estadísticas individuales en eventos cualitativos, cubriendo extracción de datos, cálculo de puntajes y visualización de datos.
  - Se actualizó `pre-poqtrol-stats-user-generator.ts` para corregir la referencia de servicio de `PeecStatService` a `PoqtrolStatService`.
* Refactorización de la documentación de módulos y procesos estadísticos en las secciones de ERP y Laboratorio:
  - Actualización de terminología de "módulos de características" a "módulos funcionales" para mayor claridad.
  - Mejora de las descripciones y organización de las funcionalidades clave en el módulo ERP.
  - Expansión del módulo de Laboratorio para incluir procesos detallados de análisis estadístico y enlaces a diagramas relevantes.
  - Nueva documentación de los procesos estadísticos de PEEC y Quattrol 365.
  - Nuevas secciones introducidas para las estadísticas de usuario en el módulo POQTROL, que detallan las evaluaciones de rendimiento individuales.
